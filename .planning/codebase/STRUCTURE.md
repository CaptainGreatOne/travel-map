## Naming Conventions

**Files:**
- PascalCase for React components: `MapPage.jsx`, `AuthModal.jsx`
- camelCase for utilities: `supabaseClient.js`, `sampleData.js`
- Page components have "Page" suffix: `MapPage`, `AboutPage`

**Directories:**
- Lowercase plural names: `pages`, `components`, `contexts`, `utils`, `data`
- No kebab-case directories

**Special Patterns:**
- `.jsx` extension for all React components
- No `index.js` barrel files

## Where to Add New Code

**New Feature/Page:**
- Primary code: `src/pages/{PageName}.jsx`
- Add route in: `src/App.jsx`
- Components: `src/components/`

**New Component:**
- Implementation: `src/components/{ComponentName}.jsx`
- No separate types directory (JavaScript, not TypeScript)

**New Route:**
- Definition: `src/App.jsx` (React Router)
- Page: `src/pages/{PageName}.jsx`

**New Context:**
- Implementation: `src/contexts/{Name}Context.jsx`
- Wrap in: `src/App.jsx`

**Utilities:**
- Shared helpers: `src/utils/{name}.js`
- Service clients: `src/utils/{service}Client.js`

**Static Data:**
- Mock/sample data: `src/data/{name}.js`
